True Ranks GitHub Repository Deployment
========================================

This guide covers deploying the True Ranks datapack as its own GitHub repository.
True ranks use percentile cutoffs instead of fixed point thresholds.


True rank tiers
---------------

  Dragon   - top 1%
  Rune     - top 10%
  Adamant  - top 20%
  Mithril  - top 40%
  Steel    - top 60%
  Iron     - top 80%
  Bronze   - top 100%

For each tier the script records:
  - players_qualified (count in that top percentile)
  - point_cutoff (score at the boundary rank for that percentile)
  - players_qualified_change (delta since the last run)


Repository layout
-----------------

DataPacks/ (local only)
  Walkthrough/
    True Ranks Deployment.txt   <- this file (local reference copy)
  TrueRanks/                    <- push THIS folder as a standalone GitHub repo
    rank-update.py
    requirements.txt
    Data/
    docs/
    .github/workflows/


Step 1: Create the GitHub repository
------------------------------------

1. On GitHub, create a new empty repository (example name: rsleagues-datapacks-true-ranks).
2. Do not add a README, .gitignore, or license.

From PowerShell, inside the TrueRanks folder:

  cd f:\Dev\rsleagues\DataPacks\TrueRanks
  git init
  git add .
  git commit -m "Initial true ranks datapack"
  git branch -M main
  git remote add origin https://github.com/YOUR_USER/REPO_NAME.git
  git push -u origin main


Step 2: Enable workflow permissions
-----------------------------------

Settings -> Actions -> General -> Workflow permissions -> Read and write permissions


Step 3: Verify the cron workflow
--------------------------------

Workflow file: .github/workflows/rank-update.yml
Schedule: "0,30 * * * *" (UTC, on the hour and half past)

Actions tab -> "True Rank Update" -> "Run workflow"


Step 4: Enable GitHub Pages
---------------------------

Option A (recommended): GitHub Actions deploy
  1. Settings -> Pages -> Source: "GitHub Actions"
  2. Actions -> "Deploy Pages" -> "Run workflow"

Option B: Deploy from branch
  1. Settings -> Pages -> Source: "Deploy from a branch"
  2. Branch: main, Folder: /docs


Step 5: Use fetch() from your website
-------------------------------------

Example:

  const response = await fetch(
    "https://YOUR_USER.github.io/REPO_NAME/rank-thresholds.json"
  );
  const ranks = await response.json();

JSON shape:

  {
    "logged_at": "2026-07-09T01:30:00.123456+00:00",
    "total_players": 420000,
    "total_players_change": 3100,
    "ranks": [
      {
        "name": "Dragon",
        "point_cutoff": 84500,
        "players_qualified": 4200,
        "players_qualified_change": 42
      }
    ]
  }

On the first run, change fields are null because there is no prior JSON.


Step 6: Local development
-------------------------

  pip install -r requirements.txt
  python rank-update.py


Troubleshooting
---------------

Hiscores 403 Forbidden in Actions:
  - Jagex may block GitHub-hosted runner IPs. Use a self-hosted runner if needed.

Pages 404:
  - Enable Pages in repo settings and run the Deploy Pages workflow once.
